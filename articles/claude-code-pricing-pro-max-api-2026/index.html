클로드 코드(Claude Code)를 쓸 수 있는 결제 경로는 네 가지로 갈린다. 클로드 Pro 구독, Max 5x 구독, Max 20x 구독, 그리고 클로드 API 종량제다. 세 가지는 월정액이고 한 가지는 토큰 단위로 과금된다. 같은 도구인데 결제 방식에 따라 같은 작업 비용이 몇 배씩 달라진다.

이 글은 네 경로의 가격과 한도 구조를 비교하고, 사용 패턴별로 어느 시점에 어떤 플랜이 손해로 바뀌는지 시나리오로 정리한다. 어떤 도구를 쓸지 정한 다음 단계의 질문이다 — 어떤 도구인지 헷갈린다면 클로드 채팅·코드·코워크 입문 글이 먼저다.

1. 결제 경로는 네 갈래

월정액 세 등급과 종량제 한 가지로 구성된다. 월정액은 동일한 도구를 제공하고 한도만 다르다. 한 번 사용을 시작하면 5시간 단위로 한도가 차고, 1주일 누적 한도가 별도로 적용된다. 두 한도 중 어느 쪽이든 먼저 채워지면 그 시간만큼 대기해야 한다.

중요한 점이 하나 있다. 클로드 채팅(claude.ai)과 클로드 코드 사용량은 같은 한도를 공유한다. 채팅에서 긴 문서를 분석한 직후 코드에 가서 작업을 시작하면, 채팅에서 쓴 토큰이 코드 한도에서 차감된 상태다. 같은 구독이 두 도구를 동시에 커버한다는 뜻이기도 하고, 한쪽에서 한도를 다 쓰면 다른 쪽에서도 못 쓴다는 뜻이기도 하다.

API 종량제는 구독이 아니다. 사용한 토큰에 직접 과금되고 한도 개념이 없다. 모델별 단가가 다르고, 캐시·배치 같은 기능으로 할인이 누적된다. 팀이 자동화·CI 환경에서 클로드 코드를 박을 때 쓰는 경로다.

2. Pro — $20, 입문점

Pro는 클로드 구독의 최저 유료 등급이다. 월 $20, 연간 결제 시 $200(약 17% 할인). 한 구독으로 클로드 채팅·코드·코워크·데스크톱 앱까지 모두 묶인다. 클로드 코드만 따로 사는 옵션은 없다.

한도는 두 축으로 작동한다. 5시간 단위 사용량 한도, 1주일 누적 사용량 한도. 정확한 메시지 수치는 모델·프롬프트 길이에 따라 달라 공식 안내에서도 범위로만 제시한다. 일반적인 패턴 기준으로 보면 다음과 같다.

Pro가 잘 맞는 사람은 분명하다. 사이드 프로젝트, 주말 코딩, 회사에서 하루 1~2시간 코드 보조용. 처음 클로드 코드를 써보는 단계라면 Pro에서 시작해서 한도에 부딪히는 빈도를 한 달쯤 관찰한 다음 결정하는 편이 합리적이다.

3. Max — $100·$200, Pro 한도가 막힐 때

Pro에서 한도에 자주 부딪히면 다음 선택지는 Max다. 두 단계로 갈린다. Max 5x는 월 $100으로 Pro 대비 5배 한도, Max 20x는 월 $200으로 20배 한도. 둘 다 클로드 코드 사용은 동일하고 한도만 다르다.

Max 5x는 풀타임 코딩 보조용의 표준 구간이다. 매일 3~5시간 클로드 코드를 켜놓고 쓰는 패턴이라면 Pro에서는 주 후반에 한도가 차지만 Max 5x에서는 거의 보이지 않는다. 가격은 Pro 대비 5배($20→$100)인데 한도도 5배라 비례가 그대로 유지된다.

Max 20x는 사용량을 의식하고 싶지 않은 사람용이다. 백그라운드 에이전트를 여러 개 동시에 돌리거나 하루 종일 켜놓고 자동화 작업을 하는 패턴이 여기 해당한다. 5x → 20x로 올릴 때 가격은 2배(+$100)인데 한도는 4배(5x→20x). 이 격차가 가성비의 기준이다. 5x로 한도에 닿는 빈도가 잦으면 20x가 싸지고, 그렇지 않으면 보험료에 가깝다.

5x에서 20x로 올릴 때 가격은 2배지만 한도는 4배. 한도에 닿는 빈도가 잦을수록 가성비가 좋아지는 구조다.

4. API — 종량제, 자동화·CI 전용

클로드 코드는 ANTHROPIC_API_KEY 환경변수만 있으면 구독 없이도 실행할 수 있다. 이 경우 구독 한도가 아니라 토큰 사용량 그대로 청구된다. 1인이 단순히 쓰기에는 거의 의미 없는 경로지만 두 장면에서 빛난다.

첫째, 팀이 5명을 넘길 때. 각자 Max 5x를 사면 $500/월이 누적된다. 회사 명의 API 키 하나로 묶어 사용량을 합산하면 행정도 단순하고 비용도 사용 패턴에 따라 더 저렴해질 수 있다. 둘째, GitHub Actions·GitLab CI 같은 무인 환경. 사람이 로그인할 수 없는 곳에서는 API 키가 유일한 인증 수단이다. PR 자동 리뷰, 야간 회귀 테스트, 주간 의존성 업데이트가 이 경로에 속한다.

단점은 비용 예측이 어렵다는 것이다. 토큰 단가는 정해져 있지만 한 작업이 얼마나 많은 토큰을 쓸지는 시작해서 끝내봐야 안다. 그래서 API 경로는 사용량 알림과 월간 한도 설정이 사실상 필수다.

5. API 토큰가, 모델별로 5배까지 벌어진다

API 비용 구조는 모델 선택에서 결정된다. 같은 작업이라도 어느 모델로 처리하느냐에 따라 청구액이 5배 차이가 난다.

Opus는 Haiku 대비 입력 5배·출력 5배다. 하지만 두 모델이 같은 작업을 처리하는 효율 차이는 5배가 아니다. 단순한 리팩터링·로그 분석·문서 정리는 Haiku로 충분한 경우가 많고, 복잡한 디버깅·아키텍처 설계·다단계 추론에서는 Opus가 압도적으로 효율이 좋다. 효율 차이를 토큰가 차이가 일치시키지 못하는 작업이 분명히 있다.

여기에 두 종류의 할인이 누적된다. 둘 다 자동화 파이프라인의 비용 구조를 결정한다.

프롬프트 캐싱 — 캐시 적중 시 표준 입력가의 0.1배
긴 시스템 프롬프트, CLAUDE.md, 큰 파일 같은 반복 입력에 적용. 같은 컨텍스트가 두 번 이상 들어가면 두 번째부터 90% 절감. 5분·1시간 두 가지 TTL을 고를 수 있고 길이가 길수록 쓰기 비용은 조금 더 들지만 한 번이라도 적중하면 본전을 넘는다.
배치 API — 입출력 50% 할인
24시간 내 처리 보장의 비동기 큐. 사람이 실시간으로 기다리지 않아도 되는 작업(야간 분석, 회귀 검증, 대량 변환)에 적합. 캐싱과 함께 쓰면 동일 모델 표준가 대비 75% 가까이 절감되는 구조가 만들어진다.

6. 시나리오별 어느 게 싼가

같은 도구라도 사용 강도와 패턴에 따라 손익분기점이 달라진다. 네 가지 시나리오로 정리하면 다음과 같다.

A · 주말 사이드 프로젝트
Pro
주 5~10시간 사용. Pro 한도에 거의 안 닿는다. 연 $200 옵션이 월 $20 결제보다 17% 저렴. 처음 1~2개월 한도 체감 후 Max로 올리는 경로가 안전.
B · 풀타임 1인 개발자
Max 5x
매일 3~5시간 코드 보조로 사용. 주 20~30시간 누적. Pro로는 매주 후반에 한도가 차지만 5x에서는 거의 보이지 않는다. 20x로 올리기 전에 5x로 충분한지 1개월 검증.
C · 풀타임 + 백그라운드 에이전트
Max 20x
동시 세션 2~3개 + 자동화 스크립트 병행. 주 40시간 이상 누적 사용량. 5x에서도 한도 도달하는 빈도가 있다면 20x가 정답. 한도 신경 안 쓰고 싶을 때 지불하는 프리미엄이 +$100/월.
D · 팀 5명 + CI 자동화
API
개인 Max 5x를 5명 사면 $500/월. 사용 패턴이 사람마다 달라 일부는 한도가 남고 일부는 모자란다. API 종량제로 묶으면 실사용량 기준 정산. CI 봇용 별도 키도 같은 계정에서 발급. 캐시·배치 적용 시 균등 분배 대비 30~50% 절감 가능.

월 청구액만 보면 Pro < Max 5x < Max 20x < (사용량 많은) API 순으로 정렬되지만 단순 가격 비교는 의미가 없다. 손익분기점은 한도에 부딪히는 빈도가 결정한다. 한 번도 안 부딪힌 사람이 Max 20x를 유지하는 것과 매주 부딪히는 사람이 Pro를 유지하는 것 — 둘 다 비용 최적이 아니다.

한 번도 한도에 닿은 적 없으면서 Max 20x를 유지하는 건 보험이 아니라 낭비다. 한 단계 내려보고 막히면 그때 올리는 편이 보통은 싸다.

Q렌즈의 시각

월정액 세 등급은 "쓸 만큼 쓰는 시간"을 세 단계로 끊은 것에 가깝다. Pro는 가끔, Max 5x는 매일, Max 20x는 한도를 신경 끄고 싶은 사람. 다른 SaaS의 등급 구조와 다르지 않다. 새로운 건 등급 위로 한 칸 더 있는 API 경로다.

구독이 도구를 빌리는 것이라면 API는 원료를 사다 직접 가공하는 쪽에 가깝다. 1인에게는 거의 의미 없지만, 팀이 합쳐 5명이 되는 순간 손익이 뒤집힌다. 그 지점이 정확히 어디인지는 팀의 사용 패턴이 정한다 — 누구는 매일 4시간, 누구는 주 2시간 식의 편차가 크면 API가 빨리 싸진다.

가장 큰 함정은 "Max 20x를 사두면 안심된다"는 사고다. 안심은 월 $200어치인가, 라는 질문이 항상 따라온다. 한 번도 한도에 닿은 적 없는 사람이 20x를 유지하는 건 보험이 아니라 매몰 비용이다. 한 단계 내려보고 막히는 시점에 올리는 편이 보통은 싸다. 단계 간 차이가 한 달치 정도면 "위로 갔다 내려오는 비용"도 크지 않다.
